Cores/spacing extraídos via get_variable_defs + get_design_context (Figma Dev Mode), sempre escopado direto na instância. Esta tela usa uma paleta de cinzas diferente das telas documentadas anteriormente (#404040/#737373/#d4d4d4 em vez de #262626/#a3a3a3) — reproduzido fielmente, é uma variação real entre capturas do Figma em datas diferentes, não um erro nosso. Os ícones de "Sobre o Zupper", "Política de privacidade" e "Termos e condições" ficam atrás de instance-swap no Figma — o export vetorial direto vinha vazio (só a "Central de ajuda" resolveu de primeira); usamos download_assets escopado na linha inteira pra obter o SVG já flatten com o ícone real resolvido. Na variante Parceiro, as 3 linhas de Comunidade sem tela de destino (Posts curtidos / Preferências de viagem / Contas bloqueadas) têm anotações de pendência escritas pelo próprio time de design no Figma (visíveis no atributo title ao passar o mouse sobre a linha) — confirma que o discovery/design dessas telas (épico KSA-13) ainda não foi feito; só "Minhas publicações" redireciona pra uma tela que já existe (o perfil, ver ../comunidade/). No estado Deslogado, o ícone de "Promoções" no bottom nav veio vazio no Figma (instance-swap não resolvido, mesmo padrão de gaps já visto em outras telas) — usamos aqui o ícone de "fire" já existente no design system como substituto visual, só pra manter a leitura da tela; não é o ícone real do Figma.
